<h2 id="converter">Converter</h2>

<div class="language-java highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="kd">class</span> <span class="nc">Converter</span><span class="o">{</span>
    <span class="nc">Object</span> <span class="nf">getAsObject</span><span class="o">(</span><span class="nc">FacesContext</span><span class="o">,</span> <span class="nc">UIComponent</span><span class="o">,</span> <span class="nc">String</span><span class="o">);</span>
    <span class="nc">String</span> <span class="nf">getAsString</span><span class="o">(</span><span class="nc">FacesContext</span><span class="o">,</span> <span class="nc">UIComponent</span><span class="o">,</span> <span class="nc">Object</span><span class="o">);</span>
<span class="o">}</span>
</code></pre></div></div>

<p>视图中的每一个UIComponent都可以有0-1个Converter<br />
Converter承担一个html里的字面字符串和java对象之间的桥梁,桥梁的另一头是String,另一头可以是任意的Java类<br />
例如:<code class="language-plaintext highlighter-rouge">&lt;h:inputText ... value="#{myBean.myAttr}" /&gt;</code>,而myBean.myAttr是一个Integer,则这时候就需要IntegerConverter<br />
对于一些常见的Java类,Jsf会自动启用一些Converter,例如IntegerConverter,DoubleConverter等等</p>

<p>所有的Converter都可以通过把标签<code class="language-plaintext highlighter-rouge">&lt;f:converter converterId="..." /&gt;</code>嵌套在某个UIComponent标签内部实现,例如<code class="language-plaintext highlighter-rouge">&lt;h:inputText&gt;&lt;f:converter ... /&gt;&lt;/h:inputText&gt;</code>
当然也可以编程式: <code class="language-plaintext highlighter-rouge">FacesContext.getCurrentInstance().getViewRoot().findComponent("...").setConverter(FacesContext.getCurrentInstance().getApplication().createConverter(Integer.class));</code></p>

<blockquote>

  <p>定制化的步骤:</p>
  <ol>
    <li>实现Converter接口</li>
    <li>在faces-config.xml里配置一个<code class="language-plaintext highlighter-rouge">&lt;converter /&gt;</code>或者使用<code class="language-plaintext highlighter-rouge">@FacesConverter</code></li>
  </ol>
</blockquote>

<h2 id="composite-tag">Composite Tag</h2>

<p>jsf页面的根节点为UIViewRoot元素,类似于html里的<code class="language-plaintext highlighter-rouge">&lt;html&gt;</code><br />
所有的子元素以树状结构组织起来(<code class="language-plaintext highlighter-rouge">UIComponent.getChildern()和UIComponent.setParent(...)</code>)</p>

<p>每个UIComponent之下可以嵌套0-1个Converter,可以嵌套0-n个Validator,0-n个Listener</p>

<p>假设一种场景,一个页面引用一个组合组件:</p>

<div class="language-xml highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="nt">&lt;html</span> <span class="na">xmlns:vt=</span><span class="s">"http://java.sun.com/jsf/composite/Xxx"</span><span class="nt">&gt;</span>
    <span class="nt">&lt;h:form&gt;</span>
        <span class="nt">&lt;vt:loginPanel</span> <span class="err">...</span><span class="nt">&gt;</span>
        <span class="err">&lt;</span>/&gt;
    <span class="err">&lt;</span>/&gt;
<span class="err">&lt;</span>/&gt;
</code></pre></div></div>

<p>…/Xxx/loginPanel.xhtml</p>
<div class="language-xml highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="nt">&lt;html</span> <span class="na">xmlns:cc=</span><span class="s">"http://java.sun.com/jsf/composite"</span> <span class="err">...</span><span class="nt">&gt;</span>
    <span class="nt">&lt;cc:interface&gt;</span>
        <span class="nt">&lt;cc:attribute</span> <span class="na">name=</span><span class="s">"attr1"</span> <span class="na">default=</span><span class="s">"..."</span> <span class="nt">/&gt;</span>
        <span class="nt">&lt;cc:editableValueHolder</span> <span class="na">name=</span><span class="s">"attr2"</span> <span class="nt">/&gt;</span>
        <span class="nt">&lt;cc:actionSource</span> <span class="na">name=</span><span class="s">"attr3"</span> <span class="nt">/&gt;</span>
    <span class="err">&lt;</span>/&gt;
    <span class="nt">&lt;cc:implementation&gt;</span>
        <span class="nt">&lt;h:someTag</span> <span class="na">value=</span><span class="s">"#{cc.attrs.attr1}"</span><span class="nt">&gt;</span>
    <span class="err">&lt;</span>/&gt;
<span class="err">&lt;</span>/&gt;
</code></pre></div></div>

<p>jsf会隐式创建名为cc的bean,并在attrs属性下创建所有在interface里的属性的键值对,可以通过<code class="language-plaintext highlighter-rouge">cc.attrs.myAttr</code>来获取<br />
当完之后,loginPanel.xml就变成了一个名为<code class="language-plaintext highlighter-rouge">loginPanel</code>的标签,如果这个时候在jsf文件里导入组合库<code class="language-plaintext highlighter-rouge">http://java.sun.com/jsf/composite/Xxx</code>(所有组合标签必须在命名空间<code class="language-plaintext highlighter-rouge">http://java.sun.com/jsf/composite/</code>下)<br />
其中Xxx为存放标签库的文件夹,默认会在以下几个文件夹查找:</p>

<blockquote>

  <ol>
    <li>resources/Xxx</li>
    <li>META-INF/resources/Xxx</li>
    <li>WEB-INF/resources/Xxx</li>
  </ol>
</blockquote>

<p>查找之后所有Xxx文件夹下的.xhtml都可以被使用<br />
使用组合标签的页面的结构如下:</p>

<blockquote>

  <ul>
    <li>UIViewRoot
      <ul>
        <li>HtmlForm
          <ul>
            <li>UINamingContainer
              <ul>
                <li>someTag</li>
              </ul>
            </li>
          </ul>
        </li>
      </ul>
    </li>
  </ul>
</blockquote>

<p>jsf会隐式的创建一个名为UINamingContainer的组件来容纳导入的组合标签</p>

<ol>
  <li>实现Renderer接口</li>
  <li>使用faces-config.xml注册<code class="language-plaintext highlighter-rouge">&lt;renderer&gt;</code>或者使用<code class="language-plaintext highlighter-rouge">@FacesRenderer</code>,或者如果是自定义的标签,那么在<code class="language-plaintext highlighter-rouge">xxx-taglib.xml</code>里会指定<code class="language-plaintext highlighter-rouge">&lt;renderer&gt;</code>,当然可以使用编程式的<code class="language-plaintext highlighter-rouge">UIComponent.setRendererType(...)</code></li>
</ol>

<p>一个UIComponent选择适当的Render会根据两个标签family和type<br />

<p>多个<code class="language-plaintext highlighter-rouge">Renderer</code>可以同属于一个<code class="language-plaintext highlighter-rouge">RenderKit</code>,用户可以使用<code class="language-plaintext highlighter-rouge">FacesContext.getCurrentInstance().getViewRoot().setRenderKitId(String)</code>进行选择适合的RenderKit</p>

<h2 id="依赖注入-ioc">依赖注入 IOC</h2>

<p>通过在<code class="language-plaintext highlighter-rouge">faces-config.xml</code>里配置bean或者使用<code class="language-plaintext highlighter-rouge">@ManagedBean(...)</code>可以将bean交给jsf管理</p>

<blockquote>

  <p>jsf的bean和spring里的bean类似,都有一定的作用域:</p>
  <ol>
    <li>request</li>
    <li>session</li>
    <li>application</li>
    <li>…</li>
  </ol>
</blockquote>

<p>借助Validator的验证功能,jsf提供一系列helper来协助开发,例如验证一个值是不是email格式等等,使用方法直接在bean的相关对象上加上<code class="language-plaintext highlighter-rouge">@Email</code>实现:</p>

<div class="language-java highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="nd">@Contraint</span><span class="o">(...</span><span class="na">EmailValidator</span><span class="o">)</span>

<h2 id="跳转-forward--redirect">跳转 Forward / Redirect</h2>

<p>jsf里实现按钮跳转到另一个页面的步骤:</p>

<div class="language-xml highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="nt">&lt;h:commandButton</span> <span class="na">action=</span><span class="s">"#{myBean.dest()}"</span> <span class="na">value=</span><span class="s">"..."</span> <span class="nt">/&gt;</span>

<span class="c">&lt;!--==============================================--&gt;</span>

<span class="nt">&lt;h:commandLink</span> <span class="na">action=</span><span class="s">"#{myBean.dest()}"</span> <span class="na">value=</span><span class="s">"..."</span><span class="nt">&gt;</span>
    <span class="nt">&lt;f:param</span> <span class="na">name=</span><span class="s">"..."</span> <span class="na">value=</span><span class="s">"..."</span><span class="nt">&gt;</span>
<span class="nt">&lt;/h:commandLink&gt;</span>
</code></pre></div></div>

<div class="language-java highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="nd">@ManagedBean</span><span class="o">(</span><span class="n">name</span><span class="o">=</span><span class="s">"myBean"</span><span class="o">){</span>
    <span class="n">dest</span><span class="o">(){</span>

    <span class="o">}</span>
<span class="o">}</span>
</code></pre></div></div>

<div class="language-xml highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="nt">&lt;faces-config&gt;</span>
    <span class="nt">&lt;navigation-rule&gt;</span>
        <span class="nt">&lt;from-view-id&gt;</span>first-page.jsp<span class="nt">&lt;/from-view-id&gt;</span>

<p>navigation-case的from-outcome指定了from-action的返回值,如果方法执行的返回值对应了outcome的值,则跳转到second-page-jsp页面</p>

<h2 id="生命周期回调-lifecycle-callback">生命周期回调 Lifecycle Callback</h2>

<blockquote>

  <p>jsf的应用分为六个主要的生命周期:</p>
  <ol>
    <li>Restore View</li>
    <li>Apply Request Values</li>
    <li>Process Validations</li>
    <li>Update Model Values</li>
    <li>Invoke Application</li>
    <li>Render Response</li>
  </ol>
</blockquote>

<blockquote>

  <p>每个<code class="language-plaintext highlighter-rouge">UIComponent</code>带有六个<code class="language-plaintext highlighter-rouge">processXxx</code>函数会在特定的生命周期被调用:</p>
  <ol>
    <li>UIComponent.processRestoreState(FacesContext, Object)</li>
    <li>UIComponent.processDecode(FacesContext)</li>
    <li>UIComponent.processValidators(FacesContext)</li>
    <li>UIComponent.processUpdates(FacesContext)</li>
    <li>UIViewRoot.processApplication(FacesContext)</li>
    <li>UIComponent.processSaveState(FacesContext)</li>
  </ol>
</blockquote>

<p>每个生命周期的后面(除了Render Response)都有一个processEvents周期,会调用UIComponent.processEvent(FacesContext)<br />
每个周期中如果调用了<code class="language-plaintext highlighter-rouge">facesContext.renerResponse()</code>或者<code class="language-plaintext highlighter-rouge">facesContext.responseComplete()</code>都会跳过中间过程</p>

<blockquote>

  <p>对于jsf应用请求分为两种:</p>
  <ol>
    <li>initial request : 第一次请求jsf页面的时候</li>
    <li>postback request : 在这个页面通过submit按钮提交的请求</li>
  </ol>
</blockquote>

<p>initial request阶段只执行1和6生命周期(restore view &amp; render response),jsf在1周期之后会调用<code class="language-plaintext highlighter-rouge">FacesCoontext.getCurrentInstance().renderResponse()</code>(注意每个生命周期之后都有一个周期回调,会调用UIComponent.processEvent(…),直接中间步骤直接渲染的功能就是通过在UIComponent.processEvent(…)里直接用FacesContext.getCurrentInstance().renderResponse()实现的)<br />
如果请求的数据是静态文件(图片等等),则直接调用<code class="language-plaintext highlighter-rouge">FacesContext.getCurrentInstance().responseComplete()</code>直接免去渲染过程</p>

<h2 id="消息系统-facesmessage">消息系统 FacesMessage</h2>

<blockquote>

  <p>类似日志,jsf的日志有三个部分组成:</p>
  <ol>
    <li>严重性(serverity)</li>
    <li>摘要(summary)</li>
    <li>细节(detail)</li>
  </ol>
</blockquote>

<blockquote>

  <p>其中serverity有四个等级:</p>
  <ol>
    <li>FacesMessage.SERVERITY_FATAL</li>
    <li>FacesMessage.SERVERITY_ERROR</li>
    <li>FacesMessage.SERVERITY_WARN</li>
    <li>FacesMessage.SERVERITY_INFO</li>
  </ol>
</blockquote>

<p>重要性依次递减,FacesMessage的使用场景如下:<br />
设想一个按钮<code class="language-plaintext highlighter-rouge">&lt;h:commandButton action="myBean.myMethod()"&gt;</code></p>

<div class="language-java highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="nc">String</span> <span class="nf">myMethod</span><span class="o">(){</span>
    <span class="nc">FacesContext</span><span class="o">.</span><span class="na">getCurrentInstance</span><span class="o">().</span><span class="na">addMessage</span><span class="o">(</span><span class="kc">null</span><span class="o">,</span> <span class="k">new</span> <span class="nc">FacesMessage</span><span class="o">(</span><span class="nc">FacesMessage</span><span class="o">.</span><span class="na">SEVERITY_ERROR</span><span class="o">,</span> <span class="s">"summary1"</span><span class="o">,</span> <span class="s">"detial1"</span><span class="o">));</span>
    <span class="nc">FacesContext</span><span class="o">.</span><span class="na">getCurrentInstance</span><span class="o">().</span><span class="na">addMessage</span><span class="o">(</span><span class="kc">null</span><span class="o">,</span> <span class="k">new</span> <span class="nc">FacesMessage</span><span class="o">(</span><span class="nc">FacesMessage</span><span class="o">.</span><span class="na">SEVERITY_INFO</span><span class="o">,</span> <span class="s">"summary2"</span><span class="o">,</span> <span class="s">"detial3"</span><span class="o">));</span>

    <span class="nc">FacesMessage</span><span class="o">.</span><span class="na">Serverity</span> <span class="n">s</span> <span class="o">=</span> <span class="nc">FacesContext</span><span class="o">.</span><span class="na">getCurrentInstance</span><span class="o">().</span><span class="na">getMaxiumSeverity</span><span class="o">();</span><span class="c1">//当前消息队列里重要性最高的是什么</span>

    <span class="k">if</span><span class="o">(</span><span class="n">s</span><span class="o">.</span><span class="na">equal</span><span class="o">(...)){</span>
        <span class="o">...</span>
    <span class="o">}</span>
    <span class="k">else</span><span class="o">{</span>
        <span class="o">...</span>
    <span class="o">}</span>
<span class="o">}</span>
</code></pre></div></div>

<p>事实上,所有的converter和validator出现的错误,都会产生一条FacesMessage(ValidatorException和ConverterException接收FacesMessage作为参数)<br />
jsf提供了<code class="language-plaintext highlighter-rouge">&lt;h:message&gt;</code>和<code class="language-plaintext highlighter-rouge">&lt;h:messages&gt;</code>来打印出消息</p>

<p>用户在<code class="language-plaintext highlighter-rouge">&lt;application&gt;</code>标签下的<code class="language-plaintext highlighter-rouge">&lt;message-bundle&gt;</code>指定一个<code class="language-plaintext highlighter-rouge">.properties</code>文件,文件内包含了键值对,用来对错误信息的字符串部分进行替换:</p>

<div class="language-java highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="cm">/**
 * 假设默认情况下XxxValidator验证错误出现FacesMessage: Xxx
 * 但是具体看,举个例子,看LengthValidator的validate方法:
 */</span>
<span class="kd">public</span> <span class="kt">void</span> <span class="nf">validate</span><span class="o">(</span><span class="nc">FacesContext</span> <span class="n">context</span><span class="o">,</span> <span class="nc">UIComponent</span> <span class="n">component</span><span class="o">,</span> <span class="nc">Object</span> <span class="n">value</span><span class="o">)</span> <span class="kd">throws</span> <span class="nc">ValidatorException</span><span class="o">{</span>
    <span class="o">...</span>
    <span class="k">throw</span> <span class="k">new</span> <span class="nf">ValidatorException</span><span class="o">(</span><span class="nc">MessageFactory</span><span class="o">.</span><span class="na">getMessage</span><span class="o">(</span><span class="n">context</span><span class="o">,</span> <span class="no">MAXIMUM_MESSAGE_ID</span><span class="o">,</span> <span class="o">...));</span>
    <span class="cm">/**
     * 可以看到抛出的信息使用了MAXIMUM_MESSAGE_ID
     * 这是一个写死在类里头的static变量: public static final String MAXIMUM_MESSAGE_ID = "javax.faces.validator.LengthValidator.MAXIMUM";
     * 运行时会根据ResourceBundle进行本地化(localize)处理,根据的就是faces-config.xml里的message-bundle
     * 根据message-bundle指定的.properties文件的键值对,然后对匹配的字符串进行替换
     * 例如.properties文件里存在 ABC=123, 则本地化之后会把要进行本地化的字符串“ABC"替换成"123"
     */</span>
<span class="o">}</span>
</code></pre></div></div>

<p>更具体的说来,本地化借助<code class="language-plaintext highlighter-rouge">java.util.ResourceBundle</code>来进行本地化,<code class="language-plaintext highlighter-rouge">FacesContext.getCurrentInstance().getApplication().getResourceBundle()</code>返回的字符串作为<code class="language-plaintext highlighter-rouge">ResourceBundle.getBundle(...)</code>的输入获取一个<code class="language-plaintext highlighter-rouge">ResourceBundle</code><br />
如果<code class="language-plaintext highlighter-rouge">application.getResourceBundle()</code>为null,即没有绑定值,则会去查找<code class="language-plaintext highlighter-rouge">faces-config</code>的<code class="language-plaintext highlighter-rouge">&lt;&lt;message-bundle&gt;</code>的值,若还是没有绑定,则默认为<code class="language-plaintext highlighter-rouge">javax.faces.Messages</code>字符串作为<code class="language-plaintext highlighter-rouge">ResourceBundle.getBundle("javax.faces.Message")</code>的输入<br />
对于FacesMessage,将在ResourceBundle里查询其对应的Id,例如<code class="language-plaintext highlighter-rouge">MINIMUM_MESSAGE_ID</code>,在ResourceBundle对应的<code class="language-plaintext highlighter-rouge">.properties</code>文件里查找所对应的值(<code class="language-plaintext highlighter-rouge">MINIUM_MESSAGE_ID=123123</code>则为<code class="language-plaintext highlighter-rouge">123123</code>)</p>

<p>同时,如果在Id后面加上<code class="language-plaintext highlighter-rouge">_summary</code>和<code class="language-plaintext highlighter-rouge">_detail</code>就会作为对应FacesMessage的summary和detail(<code class="language-plaintext highlighter-rouge">MINIUM_MESSAGE_ID_summary</code>和<code class="language-plaintext highlighter-rouge">MINIUM_MESSAGE_ID_detail</code>)<br />
本地化之后的字符串可能是<code class="language-plaintext highlighter-rouge">abc {0} def {1}</code>,这个时候对于参数<code class="language-plaintext highlighter-rouge">{0}{1}</code>可以使用<code class="language-plaintext highlighter-rouge">java.text.MessageFormat</code>进行转换</p>
