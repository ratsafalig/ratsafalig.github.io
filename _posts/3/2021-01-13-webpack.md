<div class="article__content" itemprop="articleBody"><p>使用<code class="language-plaintext highlighter-rouge">npm</code>的<code class="language-plaintext highlighter-rouge">scripts</code>执行<code class="language-plaintext highlighter-rouge">webpack</code>,配置如下:</p>

<div class="language-json highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="p">{</span><span class="w">
  </span><span class="nl">"dependencies"</span><span class="p">:</span><span class="w"> </span><span class="p">{</span><span class="w">
    </span><span class="nl">"webpack"</span><span class="p">:</span><span class="w"> </span><span class="s2">"^5.13.0"</span><span class="p">,</span><span class="w">
    </span><span class="nl">"webpack-cli"</span><span class="p">:</span><span class="w"> </span><span class="s2">"^4.3.1"</span><span class="w">
  </span><span class="p">},</span><span class="w">
  </span><span class="nl">"scripts"</span><span class="p">:</span><span class="w"> </span><span class="p">{</span><span class="w">
    </span><span class="nl">"webpack"</span><span class="p">:</span><span class="w"> </span><span class="s2">"webpack-cli -c ./webpack.config.js"</span><span class="p">,</span><span class="w">
    </span><span class="nl">"help"</span><span class="w"> </span><span class="p">:</span><span class="w"> </span><span class="s2">"webpack-cli --help"</span><span class="w">
  </span><span class="p">}</span><span class="w">
</span><span class="p">}</span><span class="w">
</span></code></pre></div></div>

<p>注意不能使用:</p>

<div class="language-json highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="p">{</span><span class="w">
    </span><span class="err">...</span><span class="w">
    </span><span class="nl">"scripts"</span><span class="w"> </span><span class="p">:</span><span class="w"> </span><span class="p">{</span><span class="w">

    </span><span class="p">}</span><span class="w">
</span><span class="p">}</span><span class="w">
</span></code></pre></div></div>

<p>这样的话: <code class="language-plaintext highlighter-rouge">npm run webpack -c ./webpack.config.js</code>后面的参数不会被识别,而是会被识别成<code class="language-plaintext highlighter-rouge">npm run</code>的参数</p>

<h1 id="概念">概念</h1>

<p><a href="https://webpack.docschina.org/concepts/">概念</a></p>

<h2 id="entry--output">entry / output</h2>

<p>入口点的配置有几种组合:</p>
<div class="language-js highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="nx">module</span><span class="p">.</span><span class="nx">exports</span> <span class="o">=</span><span class="p">{</span>
    <span class="na">entry</span> <span class="p">:</span> <span class="dl">"</span><span class="s2">main.js</span><span class="dl">"</span>
<span class="p">}</span>
<span class="cm">/***********************output**********************/</span>
<span class="cm">/**
 * 假设:
 * main.js : alert("main")
 * 输出:
 * /main.js : alert("main") 
 */</span>
</code></pre></div></div>
<div class="language-js highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="nx">module</span><span class="p">.</span><span class="nx">exports</span> <span class="o">=</span> <span class="p">{</span>
    <span class="nx">entry</span> <span class="o">=</span> <span class="p">[</span>
        <span class="dl">"</span><span class="s2">main.js</span><span class="dl">"</span><span class="p">,</span>
        <span class="dl">"</span><span class="s2">main2.js</span><span class="dl">"</span>
    <span class="p">]</span>
<span class="p">}</span>
<span class="cm">/***********************output**********************/</span>
<span class="cm">/**
 * 假设:
 * main.js : alert("main")
 * main2.js : alert("main2")
 * 输出:
 * /main.js : alert("main");alert("main2")
 */</span>
</code></pre></div></div>
<div class="language-js highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="nx">module</span><span class="p">.</span><span class="nx">exports</span> <span class="o">=</span> <span class="p">{</span>
    <span class="nx">entry</span> <span class="o">=</span> <span class="p">{</span>
        <span class="na">main</span> <span class="p">:</span> <span class="dl">"</span><span class="s2">main.js</span><span class="dl">"</span>
        <span class="na">main2</span> <span class="p">:</span> <span class="dl">"</span><span class="s2">main2.js</span><span class="dl">"</span>
    <span class="p">}</span>
<span class="p">}</span>
<span class="cm">/***********************output**********************/</span>
<span class="cm">/**
 * 假设:
 * main.js : alert("main")
 * main2.js : alert("main2")
 * 输出:  
 * /main.js : alert("main")
 * /main2.js : alert("main2")
 */</span>
</code></pre></div></div>

<p>所以,以数组形式组织的entry,是按entry里的顺序把所有的入口合并成同一个入口<br />
而按对象形式组织的entry,是按entry里的键值拆分成多个入口</p>

<h1 id="定制化">定制化</h1>

<h2 id="loader">loader</h2>

<p><a href="https://webpack.docschina.org/loaders/">loaders</a></p>

<p><a href="https://webpack.docschina.org/contribute/writing-a-loader/">write-a-loader</a></p>

<p>js里,正则表达式字符串的定义方式为:  <code class="language-plaintext highlighter-rouge">/.../</code>,在两个正斜杠中间的为正则表达式内容</p>

<div class="language-js highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="nx">module</span><span class="p">.</span><span class="nx">exports</span> <span class="o">=</span> <span class="p">{</span>
  <span class="c1">//...</span>
  <span class="na">module</span><span class="p">:</span> <span class="p">{</span>
    <span class="na">rules</span><span class="p">:</span> <span class="p">[</span>
      <span class="p">{</span>
        <span class="na">test</span><span class="p">:</span> <span class="sr">/</span><span class="se">\.</span><span class="sr">js/</span><span class="p">,</span>
        <span class="na">use</span><span class="p">:</span> <span class="p">[</span>
          <span class="dl">'</span><span class="s1">bar-loader</span><span class="dl">'</span><span class="p">,</span>
          <span class="dl">'</span><span class="s1">foo-loader</span><span class="dl">'</span>
        <span class="p">]</span>
      <span class="p">}</span>
    <span class="p">]</span>
  <span class="p">}</span>
<span class="p">}</span>
</code></pre></div></div>
<p>如上的配置会对依赖图里的所有匹配<code class="language-plaintext highlighter-rouge">\.js</code>字符串启用<code class="language-plaintext highlighter-rouge">foo-loader</code>,然后把输出再传递给<code class="language-plaintext highlighter-rouge">bar-loader</code><br />
loader的加载顺序为从下至上,最底下的loader最先调用</p>

<p>模块还可以使用一种叫做<code class="language-plaintext highlighter-rouge">inline</code>的方式调用:</p>

<div class="language-js highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="k">import</span> <span class="nx">Styles</span> <span class="k">from</span> <span class="dl">'</span><span class="s1">style-loader!css-loader?modules!./styles.css</span><span class="dl">'</span><span class="p">;</span>
</code></pre></div></div>

<p>同样是靠后的loader最先执行,然后按链式传递给上一个loader<br />
加载中可能出现一些<code class="language-plaintext highlighter-rouge">!</code>或是<code class="language-plaintext highlighter-rouge">!!</code>:</p>

<p><a href="https://webpack.js.org/concepts/loaders/#inline">loader#inline</a></p>

<p>loader是一个简单的函数:</p>

<div class="language-js highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="k">export</span> <span class="k">default</span> <span class="kd">function</span><span class="p">(</span><span class="nx">source</span><span class="p">)</span> <span class="p">{</span>
    <span class="p">...</span>
<span class="p">}</span>
</code></pre></div></div>

<h2 id="plugin">plugin</h2>

<p><a href="https://webpack.docschina.org/contribute/writing-a-plugin/">write-a-plugin</a></p>

<p><a href="https://webpack.docschina.org/plugins/">plugins</a></p>

<p><a href="https://github.com/jantimon/html-webpack-plugin">html-webpack-plugin</a></p>

<div class="language-js highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="kd">class</span> <span class="nx">MyPlugin</span><span class="p">{</span>
    <span class="nx">apply</span><span class="p">(</span><span class="nx">compiler</span><span class="p">){</span>
        
    <span class="p">}</span>
<span class="p">}</span>
</code></pre></div></div>

<p>plugin是一个具有<code class="language-plaintext highlighter-rouge">apply(compiler)</code>的类,<code class="language-plaintext highlighter-rouge">compiler</code>具有编译过程的一系列回调属性,可以通过将回调函数写入<code class="language-plaintext highlighter-rouge">compiler</code>的一系列属性达到在特定生命周期进行回调的效果:</p>

<div class="language-js highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="kd">class</span> <span class="nx">MyPlugin</span><span class="p">{</span>
    <span class="nx">apply</span><span class="p">(</span><span class="nx">compiler</span><span class="p">){</span>
        <span class="nx">compiler</span><span class="p">.</span><span class="nx">hooks</span><span class="p">.</span><span class="nx">compilation</span><span class="p">.</span><span class="nx">tap</span><span class="p">(</span><span class="dl">"</span><span class="s2">...</span><span class="dl">"</span><span class="p">,</span> <span class="nx">compilation</span><span class="o">=&gt;</span><span class="p">{...})</span>
        <span class="nx">compiler</span><span class="p">.</span><span class="nx">hooks</span><span class="p">.</span><span class="nx">optimize</span><span class="p">.</span><span class="nx">tap</span><span class="p">(</span><span class="dl">"</span><span class="s2">...</span><span class="dl">"</span><span class="p">,</span> <span class="nx">compilation</span><span class="o">=&gt;</span><span class="p">{...})</span>
        <span class="nx">compiler</span><span class="p">.</span><span class="nx">hooks</span><span class="p">.</span><span class="nx">emit</span><span class="p">.</span><span class="nx">tapAsync</span><span class="p">(</span><span class="dl">"</span><span class="s2">...</span><span class="dl">"</span><span class="p">,</span> <span class="p">(</span><span class="nx">compilation</span><span class="p">,</span> <span class="nx">callback</span><span class="p">)</span><span class="o">=&gt;</span><span class="p">{...})</span>
        <span class="nx">compiler</span><span class="p">.</span><span class="nx">hooks</span><span class="p">.</span><span class="nx">emit</span><span class="p">.</span><span class="nx">tapPromise</span><span class="p">(</span><span class="dl">"</span><span class="s2">...</span><span class="dl">"</span><span class="p">,</span> <span class="nx">compilation</span><span class="o">=&gt;</span><span class="p">{...})</span>
    <span class="p">}</span>
<span class="p">}</span>
</code></pre></div></div>

<h2 id="代码分离">代码分离</h2>

<p><a href="https://webpack.docschina.org/guides/code-splitting/">code-splitting</a></p>

<p>设想场景,入口 A 依赖 B , B 依赖 C ,….<br />
如果浏览器在需要 A 的时候直接把 A 的所有依赖都加载进来,会造成明显的停顿,影响用户体验<br />
为了改善现象, webpack 提供代码分离功能,在使用到某一部分的时候才会请求这一个部分的 js<br />
这一个个部分叫做一个 chunk</p>
